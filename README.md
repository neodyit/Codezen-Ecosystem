# CodeZen Ecosystem

🔗 **Live Website:** [https://codezen.neodyit.in](https://codezen.neodyit.in)

CodeZen is CGC University's student-led technology community empowering students through technical excellence, leadership, innovation, and industry-ready skills. This repository contains the source code for the CodeZen Ecosystem web application.

## 🚀 Quick Start

To get started with the project locally:

1. **Clone the repository**
   ```bash
   git clone https://github.com/neodyit/Codezen-Ecosystem.git
   cd Codezen-Ecosystem
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Run the development server**
   ```bash
   npm run dev
   ```

4. Open your browser and visit `http://localhost:5173`

## 🛠️ Technology Stack
- **Framework:** React with Vite
- **Styling:** Tailwind CSS & Custom Vanilla CSS
- **Animations:** Framer Motion & GSAP
- **Routing:** React Router DOM
- **Smooth Scrolling:** Lenis

## 📁 Project Structure
- `src/pages/`: Contains all routing pages organized by domain (`public`, `auth`, `dashboard`, `errors`).
- `src/components/ui/`: Reusable, generic UI components and interactive elements.
- `src/components/sections/`: Larger block sections that make up the pages.

## 🤝 How to Collaborate

We welcome contributions from everyone in the community! If you're looking to help out, please follow these steps:

### 1. Find an Issue
- Check our [Issues Tracker](https://github.com/neodyit/Codezen-Ecosystem/issues) for open tasks.
- Look for issues tagged with `good first issue` or `help wanted` if you're a beginner.
- If you have a new feature idea or found a bug, please **open a new issue** to discuss it before writing code.

### 2. Fork and Branch
- Fork this repository to your own GitHub account.
- Create a new branch for your feature or bugfix. Name it descriptively, e.g., `feature/login-page` or `fix/header-alignment`.

### 3. Make Your Changes
- Ensure your code follows the existing style and architecture.
- For new UI components, place them in `src/components/ui`.
- Use **Tailwind CSS** for styling where possible, and avoid inline styles unless strictly necessary.
- Write meaningful commit messages.

### 4. Open a Pull Request (PR)
- Push your branch to your forked repository.
- Open a Pull Request against our `main` branch.
- In your PR description, explain **what** changes you made and **why**.
- Link any related issues (e.g., "Fixes #42").

### 5. Review Process
- A core team member will review your PR.
- We might suggest some changes or improvements. Don't worry, it's a collaborative process!
- Once approved, your code will be merged into the main project.

Thank you for helping us build the CodeZen Ecosystem!
